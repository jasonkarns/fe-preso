## container properties - display - flex-flow - flex-direction - flex-wrap - justify-content - align-items - align-content
``` display: flex; ``` - creates a flex container - its children are automatically flex items - can also be containers themselves
``` css flex-flow: <'flex-direction'> || <'flex-wrap'> ``` ``` css flex-direction: row | row-reverse | column | column-reverse; ``` ``` css flex-wrap: nowrap | wrap | wrap-reverse; ``` Note: - flow is shorthand - direction depends on ltr/rtl - wrap depends on direction
- alignment along *main axis* ``` css justify-content: flex-start | flex-end | center | space-between | space-around; ``` ![](img/flexbox-justify-content.svg)
- alignment along cross axis - dual of justify-content ``` css align-items: flex-start | flex-end | center | baseline | stretch; ``` ![](img/flexbox-align-items.svg)
- alignment of *lines* along main axis - dual of justify-content but for rows/columns, not items ``` css align-content: flex-start | flex-end | center | space-between | space-around | stretch; ``` ![](img/flexbox-align-content.svg)
## item properties - order - flex - flex-grow - flex-shrink - flex-basis - align-self
specify non-source order of individual items ``` css order: <integer>; ``` Note: - defaults to 0 - can be negative
control width (height) of individual items ``` css flex-grow: <number>; flex-shrink: <number>; flex-basis: <length> | auto; ``` ``` css flex: <'flex-grow'> <'flex-shrink'> <'flex-basis'>; ``` ![](img/flexbox-flex-grow.svg) Note: - flex defaults to 0, 1, auto - can't be negative
control cross-axis alignment of individual items ``` css align-self: auto | flex-start | flex-end | center | baseline | stretch; ``` ![](img/flexbox-align-self.svg)

# Media Queries
# media-type ``` html <link rel='stylesheet' media='print' href='css/print.css' /> ``` ``` css background: url(some-image.png); color: red; @media print { background: white; color: black; } ```
# media-feature ``` html <link rel='stylesheet' media='all and (min-width: 701px) and (max-width: 900px)' href='css/medium.css' /> ``` ``` css @media all and (min-width: 701px) and (max-width: 900px) { #sidebar a { /* special styles for small screen */ } } ``` Note: - only is used to block older browsers - can combine media-type + media-feature or separate
# features - height/width, device-height/device-width - aspect-ratio, device-aspect-ratio - color, color-index, monochrome - resolution - orientation Note: - min/max variants of all of them (except orientation)

# CSS Units [which to use when](http://demosthenes.info/blog/775/Which-CSS-Measurements-To-Use-When)
- pixels (px) * use for: hairline borders/box-shadows * don't use for: typography, `@media` breakpoints - percentages (%) * use for: container dimensions * don't use for: typography - em, ex * use for: typography and related (margin/padding) - rem * use for: same as em/ex, better for `@media` breakpoints
- points, picas (pt, pc) * use for: print typography/images/etc stylesheets * don't use for: anything else - inches, centimeters (in, cm) * use for: print style margins/container dimensions * don't use for: anything else
- character (ch) * use for: adjusting monospace typography - viewport height/width (vh, vw) * use for: responsive typography (headers), responsive containers * don't use for: `@media` breakpoints - unitless (raw numbers) * use for: `line-height`, `border-image`

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events

Additionally custom events can be triggered on a per slide basis by binding to the data-state name. 


    Reveal.addEventListener( 'customevent', function() {
      console.log( '"customevent" has fired' );
    } );
